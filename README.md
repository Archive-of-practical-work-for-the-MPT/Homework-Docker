# Практическая работа № 11 - Начало работы с GitLab

## Цель

Развернуть локальный GitLab, создать нового пользователя с ролью Разработчик и задать ему логин и пароль, с аккаунта нового пользователя с другого устройства запушить любой проект через консоль в GitLab, создать список с задачами и взять любую задачи в реализацию

## Как сделать чтобы работало?
- В корне вашего диска создайте папку gitlab
- В папке gitlab создайте папку opt
- В папке opt создайте opt
Затем вам нужна такая архитектура:
```
/config
/data
/gitlab-runner
/logs
/run/docker.sock
```
Возможо нужны еще какие-то папки.

Перед запуском docker-compose, убедитесь, что у вас для докера установлено около 7Гб+ RAM, иначе он может работать очень медленно, затем запустите:
- docker-compose up

После чего нужно перейти в терминал контейнера и выполнить команлды по созданию администратора (читайте в интернете как или смотрите видео-лекцию).

Перейдите по localhost:8999 сначала будет ничего, нужно подождать до тех пор, пока не появится 502, а затем уже и сама страница с авторизацией.

> Комить с другого компьютера на ваш локальный GitLab можно только находясь в одной локальной сети. Для этого подключитесь к одному WiFi, либо прокиньте соединение через Radmin VPN.
  
## Цель
1. Описать основные вкладки и функции, которые есть на гитлабе. Использовать любые 5 вкладочек, которыми вы раньше никогда не пользовались (попробовать что это, и для чего нужно).
2. Узнать, как создавать новых пользователей. Создать нового пользователя с ролью Разработчик и задать ему логин и пароль.
3. С аккаунта нового пользователя необходимо с другого устройства запушить любой проект через консоль в Gitlab.
4. Создать список с задачами. Распределить задачи на 3 группы - Сделано, В работе, Ожидание. От нового пользователя взять любую задачи в реализацию.
