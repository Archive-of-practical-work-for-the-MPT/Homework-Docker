# Практическая работа № 10 - Нагрузочное тестирование приложения c фейковыми данными

## Описание

В данной практической нужно научиться реализовывать нагрузочное тестирование с помощью Locust для нашего Web приложения и/или API. Я выбрал свой проект - [Django УП 3 курс](https://github.com/Archive-of-practical-work-for-the-MPT/Django-Educational-Practice), который был доработан Dockerfile'ом.

В данном репозитории две папки:
- locust_lection - лекция
- coralwave - практическая работа

## Как сделать чтобы работало?
- В setting.py настройки подключение к вашей БД
- python -m venv .venv
- .\venv\Scripts\activate
- python manage.py migrate
- python .\manage.py createsuperuser (запроси email и пароль)
- python manage.py runserver
- Открыть в браузере на localhost

Либо через Docker
- docker-compose up

Запуск locust:
- python -m venv .venv
- .\.venv\Scripts\activate
- pip install locust
- pip install faker
- locust -f locustfile.py
- Затем запустите Web страницу которая доступна по localhost и запустите сам тест.
  
## Цель

1. Необходимо создать нагрузочное тестирование с помощью Locust вашего Web приложения и/или API.
2. Необходимо создать минимум 5 различных task для тестов в рамках одного или нескольких классов. В одном классе должно быть минимум 2 теста.
3. Разобрать чем отличаются SequentialTaskSet и TaskSet. Подобрать подходящий вариант (или использовать оба).
4. Запросы должны быть разные, и не должны быть все просто на открытие страницы методом Get.
5. Тестовые данные должны рандомно генерироваться, а также они должны быть адекватными (библиотека Faker в помощь, можно использовать и другие библиотеки).
6. Также установить, чтобы все Get запросы выполнялись минимум в 5 раз чаще, чем Post.
7. Продемонстрировать результаты тестирования ваших запросов. Запросы не должны быть все Fail - тогда это означает, что вы запрос прописали не корректно.
8. В отчете также должны быть видны результаты тестирования в виде графиков.
